1. Disable USB Autosuspend (The "Mac/Windows" Fix)Windows and macOS often keep USB ports at full power. Linux tries to save energy, which kills these hubs.Run this command to disable autosuspend for the hub immediately:bashecho -1 | sudo tee /sys/bus/usb/devices/5-4/power/autosuspend
Use code with caution.Note: If the hub address changed from 5-4, check lsusb -t to find the new ID.2. Disable Link Power Management (LPM)This is a specific "quirk" for GenesysLogic hubs. Linux attempts to use USB3/USB2 LPM to save power on the link, which frequently causes disconnects.Create a "quirks" file:sudo nano /etc/modprobe.d/usb-quirks.confAdd this line (using your hub's ID from the logs):textoptions usbcore quirks=05e3:0610:k
Use code with caution.The k flag tells the kernel to disable LPM for this specific hub.Reboot your machine.
